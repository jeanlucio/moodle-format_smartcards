# ✨ Features

* 🃏 **Card-Based Grid:** Every activity renders as an icon + title button instead of a plain list link, reusing Moodle's native availability logic (`cm_info`) end-to-end — never a "button image + stealth activity" workaround, so restriction reasons and open/close dates are never lost.
* 🎨 **Custom Card Appearance:** Teachers can replace an activity's or a section's default icon with an uploaded image, a single emoji, or a curated Bootstrap Icons library icon — plus an independent circle background colour, title colour, and title font, each from a validated, curated palette.
* 🔒 **Native Availability Badges:** A restricted activity shows a lock badge with the exact restriction reason `core_availability` already computes; a timed activity shows a clock badge with its due date. No badge means freely accessible. Nothing is ever silently hidden — a restricted-but-visible section still appears, greyed out, with its reason shown.
* 📋 **Status Sheet:** Tapping a badged card opens a sheet with the restriction reason, due date, completion criteria (mirroring the same pill widget Moodle's own course page uses), and the activity description when the teacher enabled "Show description on course page" — all computed from data Moodle core already tracks, never recalculated.
* ✅ **Completion Badges:** A second, independent badge shows pending (⚪) or complete (✅) completion state, with a manual-completion toggle right in the status sheet when the activity uses manual tracking.
* 🧭 **Six Navigation Styles:** Choose how sections are presented — **Default** (flat grid), **Accordion** (collapsible, auto-opens the section with a pending activity), **Tabs**, **Sticky** (section headers stick while scrolling), **Section cards** (each section becomes its own card, opening a full-screen modal with its nested grid), or **Trail** (a single winding column, Duolingo-style zigzag, CSS-only).
* 🧩 **"General" Section Opt-in:** By default section 0 ("General") always renders as a plain section above whatever navigation style is active. The `generalinstyle` option lets it join the style instead — collapsible in the accordion, its own tab, its own card, or part of the trail.
* 📊 **Progress Display:** An optional progress indicator next to each section's title or tab, either as a count ("2 / 5") or a percentage ("40%") — computed from Moodle's own completion tracking, never a separate calculation.
* 🎛️ **Course-Wide Appearance Defaults:** Card size (small/medium/large), card frame on/off, and default background/title colour/font are configurable site-wide with a per-course override — an individual activity's or section's own appearance always takes priority over the course default.
* 🖼️ **Icon Fallback Chain:** A section without its own configured appearance automatically borrows the first activity's configured icon inside it, before falling back to a generic section icon — so a section-cards grid still looks intentional even before every section is customised one by one.
* 📦 **Backup & Restore:** Full Moodle 2 backup/restore support for every custom appearance (activity and section, including uploaded images), correctly remapped to the new course module/section ids and file ids on restore or course duplication.
* 🔐 **Privacy API:** Declared `null_provider` — the format stores no personal data of its own; every custom appearance is course/activity configuration, never tied to a specific student.
* ♿ **Accessibility:** The activity/section title is always real text (never drawn inside an image), so it zooms, is read by screen readers, and is localised; decorative icons/emoji carry `aria-hidden`; every interactive control has a proper accessible name.
